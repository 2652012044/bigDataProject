下面我来介绍一下本项目的系统架构。整个系统可以分为**数据采集、数据清洗存储、后端服务、深度学习模型服务、前端可视化**五个层次，我按照数据的流转顺序来讲。

---

## 一、数据采集层（图上方）

【动作：手指指向左上角"逍遥模拟器"和右上角"mitmproxy"】

首先是数据采集。我使用**逍遥模拟器**运行真实的番茄小说 APP，然后通过两条技术路线来采集数据：

第一条是**自动化控制**，我用 Python 的 **uiautomator2** 框架编写了一套自动化脚本集，可以模拟用户在 APP 上的搜索、浏览、翻页等操作，自动控制采集流程。

第二条是**网络抓包**，APP 发出的所有 HTTPS 请求会经过 **mitmproxy 中间人代理**拦截。我开发了一个自定义的 mitmproxy Addon，叫 CaptureController，它能实时捕获 APP 与服务器之间的 API 通信，并将响应数据以 JSON 格式导出保存。

【动作：手指指向中间的"data/*.json"】

这两条路线汇聚到一起，最终生成原始的 JSON 数据文件，存储在 data 目录下。这就是我们整个系统的数据源头。

---

## 二、数据清洗与存储层（图中部）

【动作：手指沿"数据清洗 导入MySQL"箭头向下移动，指向MySQL】

采集到的原始 JSON 数据并不能直接使用，需要经过 **ETL 数据清洗**处理。我用 Python 脚本对原始数据进行去重、格式标准化、缺失值处理、字段映射等操作，然后批量导入到 **MySQL 数据库**中。数据库名为 novel_db，一共设计了书籍、评论、作者、分类等多张数据表，存储了数万条结构化数据。

---

## 三、后端服务层（图中间偏右）

【动作：手指指向"Spring Boot 后端 MyBatis-Plus ORM"】

数据存储好之后，接下来是后端服务。我使用 **Spring Boot** 框架搭建后端，通过 **MyBatis-Plus** ORM 框架操作数据库，对外以 **REST API** 的形式提供服务，运行在 **8080 端口**。后端实现了包括书籍分析、情感分析、智能推荐、市场趋势、搜索、收藏等全部业务逻辑。

---

## 四、深度学习模型服务（图右下角）

【动作：手指指向右下角"Flask服务 Transformer情感分析模型"】

这里是整个系统的一大亮点。我单独部署了一个 **Python Flask 微服务**，运行在 **5001 端口**，内部加载了我自主训练的 **Transformer Encoder 情感分析模型**。这个模型基于注意力机制，能对小说评论进行二分类情感预测，判断是积极还是消极。

Spring Boot 后端通过 HTTP 请求调用这个模型服务来完成情感分析。这种架构体现了**微服务**的设计思想——Java 后端和 Python 模型服务各自独立部署，通过 API 接口松耦合通信，互不影响，也方便后续单独升级模型。

---

## 五、前端展示层（图左下角）

【动作：手指指向左下方"Vue3 前端 ECharts可视化 Element Plus"】

最后是前端展示层。我使用 **Vue 3** 框架开发了单页应用，搭配 **Element Plus** 组件库构建界面，用 **ECharts** 实现各类数据可视化图表。前端通过调用后端 8080 端口的 REST API 获取数据，展示包括书籍特征分析、情感分析、推荐系统、市场趋势等多个功能模块。

---

## 总结

【动作：手指从上往下画一条线，串联整个数据流】

总结一下，整个数据流程是这样的：从**番茄小说 APP 采集**原始数据，经过**ETL 清洗**后存入 **MySQL 数据库**，**Spring Boot 后端**从数据库取数据进行统计分析，调用 **Transformer 模型**进行深度学习情感挖掘，最终通过 **Vue 3 前端**以可视化图表的形式呈现给用户。

这就是一条完整的大数据处理链路：**采集 → 清洗 → 存储 → 分析 → 挖掘 → 可视化**。

以上就是我的系统架构介绍。

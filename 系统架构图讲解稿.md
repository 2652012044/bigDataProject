# 系统架构图讲解稿（约 1 分钟）

> 在 PPT 中展示架构图时使用此稿，建议用手或激光笔边指边讲。

---

【动作：切换到架构图 PPT 页面】

下面我来介绍一下系统的整体技术架构。

【动作：手指从左侧开始，指向模拟器部分】

我的系统分为四个核心部分。

首先是**数据采集层**，在最左边。我使用逍遥模拟器运行真实的番茄小说 APP，通过 uiautomator2 框架自动化控制 APP 进行搜索和浏览操作。APP 发出的所有 API 请求都会经过 mitmproxy 中间人代理，代理拦截并记录 API 的 JSON 响应数据，存储到本地文件中。

【动作：手指移到中间下方的数据库部分】

采集到的原始数据经过 ETL 清洗处理后，导入到 **MySQL 数据库**中，也就是我的数据存储层。数据库中设计了 8 张表，存储书籍、评论、作者、分类、标签和用户数据。

【动作：手指移到中间右侧的 Spring Boot 部分】

然后是**后端服务层**，使用 Spring Boot 框架，通过 MyBatis-Plus 操作数据库，对外提供 40 多个 RESTful API 接口，涵盖数据统计、情感分析、智能推荐等全部业务逻辑。

【动作：手指移到右下方的 Flask 模型服务】

后端旁边还有一个独立的 **Python Flask 模型服务**，运行在 5001 端口。这是我自主训练的 Transformer 情感分析模型，通过 HTTP 接口提供预测服务。Spring Boot 后端调用这个模型进行情感分析，这也体现了微服务架构的思想——各服务独立部署、通过 API 松耦合通信。

【动作：手指移到最左边的 Vue 前端部分】

最后是**前端展示层**，使用 Vue 3 框架加 ECharts 可视化库，通过调用后端 API 获取数据，渲染出 22 个功能页面和一个数据可视化大屏。

【动作：手指画一个从左到右的弧线，示意整体流程】

整个数据流程是：从 APP 采集原始数据，经过清洗存入数据库，后端进行统计分析和深度学习挖掘，最终通过前端可视化呈现给用户。这就是一个完整的大数据处理链路：**采集、存储、清洗、分析、挖掘、可视化**。

以上就是我的系统架构介绍。

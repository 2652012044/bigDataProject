## 1. 评论数据（最紧缺）

**现状**：仅 50 条评论，来自 5 本书
**需要**：至少 500-1000 条评论（覆盖 50+ 本热门书）
**用途**：情感分析、口碑评分、热点分析、大屏弹幕

**爬取方式**：
- 对 `bookInfo/` 中每本书，调用评论 API 抓取更多页
- comtJson 目录目前只有 5 本书的评论文件，需扩充
- 每本书至少抓取 3-5 页评论（每页约 20 条）

---

## 2. 作者信息（严重不足）

**现状**：仅 5 位作者
**需要**：至少 50-100 位作者
**用途**：作者排行、搜索、推荐

**爬取方式**：
- 从 BookList 中提取作者名，去查对应作者详情 API
- userInfo 目录需要更多作者的 JSON 文件

---

## 3. 书籍详情（部分缺失）

**现状**：4044 本书，但大部分来自 BookList，缺少以下字段：
- `abstract`（简介）：大部分为空
- `thumb_url`（封面图）：大部分为空
- `serial_count`（章节数）：大部分为空
- `comment_total`（评论总数）：大部分为空

**需要**：至少为 Top 100 热门书补全详情
**爬取方式**：
- 对热门书籍调用书籍详情 API
- 将结果存入 `bookInfo/` 目录
- 重新运行 import_data.py 即可更新

---

## 4. 缺少的数据维度（当前爬虫未覆盖）

| 数据 | 用途 | 优先级 |
|------|------|--------|
| 书籍更新时间/最后更新章节 | 最近更新列表、趋势分析 | 高 |
| 书籍每日/每周阅读量快照 | 热度趋势折线图 | 高 |
| 标签每日热度变化 | 关键词趋势分析 | 中 |
| 读者评分分布（1-10分各多少人） | 情感分析饼图 | 中 |
| 书籍月度新增数量 | 类型趋势面积图 | 中 |

---

## 5. 建议爬取优先级

1. **高优先**：热门书籍（阅读量 Top 100）的详情 + 评论（每本 3-5 页）
2. **高优先**：更多作者详情（从书籍列表中提取唯一作者名）
3. **中优先**：每日定时爬取书籍列表，记录阅读量变化（用于趋势图）
4. **低优先**：更多分类书单（bookListTab_* 目前只覆盖了 12 个分类）

---

## 6. 导入方式

所有新数据放入对应目录后，直接运行：
```bash
python D:\bigDataProject\database\import_data.py
```
脚本会自动处理去重和增量更新。
